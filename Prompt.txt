THE PROMPT:
"Build a production-level daily habit tracker web app. Users can create an account, log in, and manage their own habits. Each habit has a name, description, optional category (health, productivity, learning, fitness, personal), and a color label. Users check off habits daily and can see a calendar heatmap view showing their consistency over time, similar to GitHub's contribution graph. Each habit shows a current streak count, longest streak, and a completion percentage for the last 30 days.
The dashboard shows today's habits with quick check-off buttons, an overall daily completion progress bar, and a weekly overview chart. Users can archive habits without losing historical data, reorder habits by drag and drop, and set reminders with custom frequencies (daily, specific weekdays, every X days).
Include a stats page with charts showing trends over time, best performing habits, and weekly/monthly summaries. Add a dark mode toggle and make the entire app fully responsive for mobile.
The app should have a clean, modern, professional look. Use Flask for the backend, SQLite for the database, Alpine.js for interactivity, and Tailwind CSS for styling. Use proper authentication with hashed passwords and session management. Structure the project with proper separation of concerns — blueprints, models, and templates organized cleanly."
